# בנה לעצמך סקילים מול מאגר הידע

> אחרי שחיברת את ה-MCP (`ONBOARDING.md`), ה-Claude Code שלך יכול לגעת בידע הארגוני.
> **סקיל** = קיצור אישי שאתה מפעיל עם `/שם` — הוא אומר ל-Claude מתי להשתמש בכלי ה-KB ואיך.
> ככה אתה בונה לעצמך "פקודות" מותאמות בלי לחכות לאף אחד.

## איך סקיל בנוי
קובץ מרקדאון תחת `~/.claude/skills/<שם>/SKILL.md`:
```markdown
---
name: at-brief
description: סיכום מצב אוטוטיוזדיי — החלטות אחרונות + סטטוס. הפעל כשרוצים תמונת מצב מהירה.
---

כשמפעילים אותי:
1. השתמש ב-`kb_search` / `kb_list` (kind=decision, ואז kind=status) מ-autotuesday-kb.
2. קרא את הפריטים הרלוונטיים עם `kb_read`.
3. סכם בעברית: מה הוחלט לאחרונה, איפה אנחנו עומדים, מה הבא. קצר וברור.
```
זהו. עכשיו `/at-brief` נותן לך סיכום מצב מהמאגר.

## הכלל הזהב
הסקיל לא "יודע" את התוכן — הוא **מוריד** אותו דרך כלי ה-KB בזמן אמת. אתה רק כותב
*מתי* להשתמש, *באיזה* כלי, ו*איך להציג*. Claude עושה את השאר. וה-RLS דואג שתראה רק
את מה שמותר לך — הסקיל לא יכול לחשוף לך משהו שאין לך הרשאה אליו.

## 3 סקילים לדוגמה (העתק, שנה, השתמש)

### 1. `/at-find` — חיפוש חכם בידע
```markdown
---
name: at-find
description: חיפוש במאגר הידע של אוטוטיוזדיי וקבלת תשובה מנוסחת. הפעל כששואלים "מה יש לנו על X".
---
1. קח את מה שהמשתמש חיפש והרץ `kb_search` (autotuesday-kb).
2. פתח את 1-3 התוצאות הכי רלוונטיות עם `kb_read`.
3. ענה תשובה ישירה בעברית, וציין מאיזה פריט (title) זה בא.
```

### 2. `/at-decision` — תיעוד החלטה חדשה
```markdown
---
name: at-decision
description: תיעוד החלטה חדשה למאגר הארגוני. הפעל כשסיכמנו משהו וצריך לשמור.
---
1. שאל את המשתמש: מה ההחלטה? לאיזה פרויקט? (org / workspace מסוים)
2. הרץ `kb_write` (autotuesday-kb): kind=decision, visibility לפי התשובה,
   title=משפט ההחלטה, body=ההקשר + התאריך.
3. אשר מה נשמר ותן את ה-id.
```

### 3. `/at-project` — צלילה לפרויקט
```markdown
---
name: at-project
description: כל מה שיש על פרויקט מסוים (למשל הזאבים / ספונסרים). הפעל כשרוצים להתעדכן בפרויקט.
---
1. `workspaces_list` — מצא את ה-workspace שהמשתמש ביקש.
2. `kb_list` עם ה-workspace_id שלו — הבא את כל הפריטים.
3. `kb_read` על SPEC/STATUS/DECISIONS. סכם: מטרה, איפה עומדים, מה פתוח.
```

## טיפים
- **התחל קטן:** סקיל אחד שימושי > עשרה תיאורטיים.
- **תן שם ברור ל-description** — לפיו Claude מחליט מתי להפעיל אוטומטית.
- **כתיבה = אחריות:** `kb_write` משנה ידע משותף. יש גרסאות (`kb_history`/`kb_rollback`),
  אז אפשר תמיד לחזור אחורה — אבל כתוב בכוונה.
- **פרטי לעצמך:** רוצה משהו שרק אתה תראה? כתוב עם `visibility=private`.

יש רעיון לסקיל שלא בטוח איך לבנות? כתוב לנתנאל/רן — נעזור.
